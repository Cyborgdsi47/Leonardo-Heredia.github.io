5º Lugar · Hackathon CopernicusLAC 2026

SafraSense — Monitoramento Agrícola por Satélite

Equipe SafraSense · CopernicusLAC / ESA · 2026

Plataforma web full-stack que democratiza o acesso a seguros agrícolas para pequenos e médios produtores rurais. O agricultor desenha o polígono de sua propriedade no mapa, cruza com dados oficiais do CAR (Cadastro Ambiental Rural) e recebe um dashboard com o índice de vegetação NDVI calculado a partir de imagens reais do satélite Sentinel-2 da ESA. O sistema gera um laudo satelital em PDF validável por seguradoras como Swiss Re e Mapfre.

Back-end

FastAPI · Python · Copernicus Sentinel Hub · GeoPandas · NumPy
  • API REST com endpoints /buscar-cars e /ndvi em FastAPI (Python)
  • Integração com Copernicus Data Space (ESA) via sentinelhub SDK para consulta de imagens Sentinel-2 L2A
  • Leitura espacial eficiente do shapefile CAR (~3 GB) com GeoPandas + Shapely: bounding box antes da interseção completa
  • Cálculo de estatísticas NDVI (mín / med / máx) por semana com NumPy
  • Série histórica de 52 semanas para detecção antecipada de estresse hídrico

Front-end

React 19 · TypeScript · TanStack Router · Leaflet · Firebase · jsPDF
  • Dashboard de status visual: Saudável / Atenção / Emergência baseado no índice NDVI
  • Mapa interativo (Leaflet + React-Leaflet) para desenho do polígono da propriedade
  • Gráfico histórico semanal de NDVI dos últimos 12 meses (Recharts)
  • Geração de laudo em PDF para seguradoras (jsPDF)
  • Autenticação e armazenamento com Firebase (Auth + Firestore)
  • Suporte multilíngue PT / ES / EN — atende Brasil, Paraguai e Bolívia

haCARthon · ENAP / Dataprev / Serviço Florestal Brasileiro

TUPÃ — Base de Referência Ambiental

Hackathon haCARthon · ENAP / Dataprev / SFB

Plataforma web full-stack que automatiza a geração de bases de referência ambiental para imóveis rurais cadastrados no CAR (Cadastro Ambiental Rural). O sistema cruza dados de satélite Sentinel-2, hidrografia BHO/ANA e relevo SRTM/NASA com as regras do Código Florestal (Lei 12.651/2012) via geoprocessamento espacial em PostGIS, entregando diagnósticos automáticos de conformidade ambiental em minutos — sem necessidade de campo ou contratação externa.

Back-end

FastAPI · Python · PostgreSQL 17 · PostGIS · SQLAlchemy · Supabase
  • Pipeline de 5 etapas: Sentinel-2 → MapBiomas → Hidrografia → Declividade → Código Florestal
  • Geoprocessamento 100% em SQL via PostGIS (ST_Buffer, ST_Intersection, ST_Intersects)
  • Cálculo de APPs por largura de curso d'água (faixas de 30 m a 200 m) e nascentes (raio 50 m)
  • Regras do Código Florestal parametrizáveis em YAML — adaptável por estado ou país
  • Score de conformidade ambiental (0–100) e exportação GeoJSON padrão OGC/SICAR
  • PostgreSQL 17 + PostGIS hospedado no Supabase (região São Paulo)

Front-end

React 18 · TypeScript · Vite · TanStack Router/Query · Leaflet · shadcn/ui
  • Painel do Analista: seleção de CAR, disparo do pipeline e visualização das feições geradas
  • Portal do Agricultor (/portal/imovel/:car) com notificações em tempo real
  • Mapa interativo (Leaflet) com camadas coloridas por tipo de feição ambiental
  • Sistema de notificações analista→agricultor com polling a cada 30 segundos
  • Auto-fallback inteligente: usa dados mock quando o backend está indisponível, com banner de aviso
